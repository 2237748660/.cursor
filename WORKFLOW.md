## 强制阶段顺序与门禁（饼王）

**新功能 / 新活动页**（包括「只丢一个蓝湖链接」）必须按顺序推进，**默认禁止跳步直写代码**。

| 序号 | 阶段 | 准入条件 | 产出 |
|------|------|----------|------|
| G0 | 🧩 参谋 — 需求分析 | 饼王接收需求 | `docs/features/<功能slug>-req-spec.md`（或页面目录下 `req-spec.md`） |
| G1 | 🔍 透视眼 — 设计分析 | G0 完成或用户**书面确认跳过** | `design-spec.md` 或等价文档 / 蓝湖 MCP 结果已存档 |
| G2 | 素材落地 | G1 可用 | 切图进入**本仓库**且路径符合「项目信息 · 静态资源约定」，**禁止**仅以设计平台 CDN 外链作为唯一交付 |
| G3 | ⚡ 像素侠 | G0+G1+G2 满足或已登记的例外 | 分层页面：切片/底图 + DOM 文案 + 热区，**禁止**仅整张设计稿单一 `background`；须满足下文 **「设计还原硬约束」** |

**例外**：用户明确说「跳过需求，直接实现」时，饼王须在对话与下方「待确认项」登记原因，并仍建议补**一页纸最小 req-spec**（路由、接口、验收 5 条内）。

### 反偷懒规范（来自复盘，具约束力）

| 违规形态 | 流程要求 |
|----------|----------|
| 未做需求分析就写页面 | 先完成 G0；未经「跳过」指令不得开工 UI |
| 整页设计图当唯一背景 | **禁止**作为唯一实现；须分层叠图 + 可维护文案 |
| 不切图、长期外链 | 必须 G2 本地化，构建可复现 |
| 单会话「一步到位」 | 每阶段饼王输出「阶段完成报告」，**默认等待用户「继续」** |
| **透明/1×1 占位图冒充已切图** | **禁止**作为 G2/G3 完成态；须为真实导出资源或 MCP 下载图，肉眼可辨版式；否则 G2 记为阻塞，**不得**申报 G3 完成 |
| **无坐标凭感觉排版** | 有画布/MCP 标注时须在 `design-spec` 或代码中与**同一标尺**对齐；无章可循须在 `design-spec`「开放问题」登记并经产品确认，禁止静默瞎写百分比 |
| **结构臆造（叠层当流式）** | 以 G1 图层树 / MCP 输出为准（如侧条浮在头图右侧 ≠ 独占一行）；重大结构写入 `design-spec` |
| **用户可见文案与稿/req 不一致** | DOM 文案以设计稿与 `req-spec` 为准；勿用另一套业务叫法凑数 |
| **热区在装饰层之下** | 透明 `button`/热区须叠在可点区域最上，可点击性可验证 |

### 设计还原硬约束（G3 交付标准）

> 本节由「实现与稿差过大」复盘写入：**透视眼**须在 G1 存档画布尺寸与图层关系；**像素侠**交付前自检；**饼王**验收 G3 时抽查；**守门员**可拒收违规合并。

| 根因 | 禁止 | 正确做法 |
|------|------|----------|
| 占位素材 | 全页 1×1、无意义占位当正式切图 | G2：**蓝湖/ Figma MCP、`lanhu_analyze_designs` 资源映射或设计导出**落入「项目信息 · 静态资源约定」路径；图片须有真实宽高以利布局 |
| 结构错误 | 按「常识」流式排版代替标注叠层 | 以 MCP HTML/图层标注为准；`design-spec` 写清「流式区块 vs 绝对叠层」及主底图文件名 |
| 坐标漂移 | 手写「大概」`left/top/%` | 有逻辑画布（如 375×h）时统一换算（如 `% = 标注值/画布`）；与工程现有 rem/vw 策略在 `design-spec` 中写一句说明 |
| 底图缺失 | 只有零散 slice、无主底或顺序错乱 | `design-spec` 列 z-index 由低到高及 `assets` 文件名；主底与装饰切片来源可追溯 |
| 文案漂移 | 占位文案与稿面用词两套体系 | 对齐 `req-spec` + 设计标注文本 |
| 交互层 | 点心区域被 `img` 盖住 | 热区 DOM 置于对应模块最末或更高 `z-index`，透明背景 |

**G3 完成声明前置条件（建议全体项目执行）**

1. 构建通过（如 `pnpm build` / 团队等价命令）。
2. 与**设计稿**或导出的标注图在目标宽度下**并排抽查 ≥1 屏**：无整段错位、无双重文案重影（若底图与切片叠看有重影，须在 `design-spec` 记明处理方案）。
